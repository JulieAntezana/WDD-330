Questions:

Is localStorage a popular way to persist data in real world applications? Or is it just a work around when we don't want to deal with sessions, or even an actual database?

  • I don't think localStorage is just a work around for sessions or a databases which are stored on the server-side. Data and assets that are saved locally will be quicker and cheaper to download and will be usable offline.
  • Most dynamic web sites today use a database to store data on a server. They must retrieve the needed data to insert into static page templates and display the resulting HTML on the browser. Session cookies have been used to store data on the client-side. Cookies are still necessary for storing sensitive information such as session IDs and access tokens, but they are more complicated than localStorage.
  • When the data is not sensitive, localStorage is a great option for storing a limited amount of data on the client-side. We can easily use localStorage for
    • personalizing user site preferences such as custom widgets, colors, or fonts
    • persisting site activity from a previous session, such as shopping cart contents
    • saving web app documents for use offline
